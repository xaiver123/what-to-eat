# 🚀 Vercel 部署超详细教程

## 准备工作
你需要有：
1. 一个 GitHub 账号（没有的话去 https://github.com 注册一个，免费的）
2. 一个 Vercel 账号（也是免费的）

---

## 第一步：把代码上传到 GitHub

### 1.1 初始化 Git 仓库
在项目文件夹中打开 PowerShell 或命令行，依次输入以下命令：

```bash
git init
git add .
git commit -m "第一次提交：今天吃什么应用"
```

### 1.2 创建 GitHub 仓库
1. 打开浏览器访问 https://github.com/new
2. 仓库名称填：`what-to-eat`（或者你喜欢的名字）
3. 选择 `Public`（公开）或 `Private`（私有）都可以
4. **不要勾选** "Initialize this repository with a README"
5. 点击 "Create repository" 按钮

### 1.3 把代码推送到 GitHub
创建仓库后，GitHub 会给你显示一些命令，找到 "…or push an existing repository from the command line" 这部分，复制那两行命令，在 PowerShell 中执行：

类似这样（你的用户名和仓库名会不一样）：
```bash
git remote add origin https://github.com/你的用户名/what-to-eat.git
git branch -M main
git push -u origin main
```

---

## 第二步：用 Vercel 部署

### 2.1 登录 Vercel
1. 访问 https://vercel.com
2. 点击 "Sign Up" 注册账号
3. 选择用 GitHub 账号登录（最方便）

### 2.2 导入项目
1. 登录后，点击 "Add New Project"
2. 找到你刚才创建的 `what-to-eat` 仓库，点击 "Import"

### 2.3 配置环境变量（非常重要！）
1. 在 "Configure Project" 页面，找到 "Environment Variables" 部分
2. 点击 "Add"
3. Name 填：`DEEPSEEK_API_KEY`
4. Value 填：你的 API Key（就是 sk-xxxxxx 那个）
5. 点击 "Add" 按钮添加
6. 确保勾选 "Automatically expose System Environment Variables"

### 2.4 部署！
1. 确认配置无误后，点击 "Deploy" 按钮
2. 等待 1-2 分钟，Vercel 会自动部署
3. 部署成功后，会显示 "Congratulations!"，并有一个链接（类似 `https://what-to-eat-xxxxx.vercel.app`）

---

## 第三步：使用！
把 Vercel 给你的链接发给朋友，他们直接在浏览器打开就能用啦！🎉

---

## 常见问题
- **部署失败了怎么办？**：检查环境变量是否正确添加，API Key 是否正确
- **想修改代码重新部署？**：在本地修改代码，然后：
  ```bash
  git add .
  git commit -m "修改了什么什么"
  git push
  ```
  Vercel 会自动重新部署！

祝你部署顺利！🍀
